1) E' effettivamente tutto fattibile, ma un server ubuntu ha bisogno di un
po' di configurazioni per diventare un file server (oltre ai servizi FTP
dovrai settare anche le condivisioni e i protocolli SMB, NFS, ecc ,
chiaramente tutto attraverso i file conf testualmente) e alcuni passaggi ti
potrebbero non semplicissimi (gestione array RAID, permessi, gruppi utenti,
maschere creazione file, ecc).
Se vuoi avere la vita un po' più semplificata ti consiglio soluzioni come
FreeNAS (basato su FreeBDS) e OMV (OpenMediaVault, basato su Debian, un po'
più flessibile). Entrambi una volta installati hanno una configurazione via
http molto comoda.
2) Se usi un RAID0, crea anche un array secondario per il backup (o fai un
RAID10). Se ti muore disgraziatamente un disco perdi tutto. In ogni caso io
preferisco le copie asincrone, visto che i raid (anche l'1 e il 10) non ti
salvano da problemi di file system. La copia la puoi fare anche via rete su
un altra macchina. Io di solito uso rsync per questo.
3) L'accensione la puoi governare in WOL via rete; oppure la setti dal
bios; oppure fai in modo che l'OS la setti sul bios (
http://www.mythtv.org/wiki/ACPI_Wakeup), col vantaggio che puoi cambiare
parametri anche da remoto in ssh.
4) per la questione DNS con bartrosso avevamo visto la soluzione
dyndns.it(il più famoso .com è ora solo a pago) e qui:
http://dyndns.it/support/software-per-aggiornamento/ c'è il programma per
aggiornare l'ip. Col chron o anacron decidi poi quando aggiornarlo.
5) per accederci dall'esterno poi o vai via FTP oppure, come dicevi Simone,
ti configuri un VPN. Anche qui c'è openvpn, ma non è una passeggiata
all'inizio...
Ciao!
Angelo.
