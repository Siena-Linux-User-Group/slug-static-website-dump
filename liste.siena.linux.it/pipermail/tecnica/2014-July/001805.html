Ciao a tutti.
Mi è risuccesso quel problema del mancato autocomplete via ssh.
Ho provato a copiare un file da un file server OMV remoto al mio a casa: ho
copiato la chiave pubblica con il copy-ssh-id sul'host remoto, auteticato
l'operazione con la password e mi sono collegato. Ho fatto una prova di
apertura sessione ssh e funzionava. Ho provato a usare l'autocomplete e
mostrava sempre e solo i files locali.
La differenza però qui è che, incollando il percorso giusto del file
remoto, sono riuscito a copiare il file, senza errore di rete.
L'autenticazione funziona quindi, ma non vuole partire l'autocomplete.
Un dettaglio che non so se influisca o meno è che sto controllando, a sua
volta, anche il computer da cui ho fatto il pull del file con ssh: in poche
parole, il mio computer si logga in ssh con il mio serverino OMV, che a sua
volta fa un scp via ssh a quello remoto. Non so se questo "incapsulamento"
possa contribuire in qualche modo.
Mica avete qualche idea?
