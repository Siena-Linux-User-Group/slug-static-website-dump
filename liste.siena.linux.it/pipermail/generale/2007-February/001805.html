Ciao a tutti,
 sto mettendo su un PDC con Samba e database LDAP su Ubuntu Server e sto 
sperimentando problemi con l'autenticazione degli utenti da console via 
LDAP. Secondo la documentazione che ho a disposizione (tratta dal sito 
idealx.org ma è per Fedora Core 5), dovrei aggiungere nel file 
/etc/nsswitch.conf la metodologia di autenticazione. In breve il file 
originale contiene:

---------------
passwd: compat
group: compat
shadow: compat
----------------

Se lo trasformo in

----------------
passwd: compat ldap
group: compat ldap
shadow: compat ldap
----------------

intendendo "prova con l'autenticazione locale, sennò con quella LDAP", 
la macchina "stenta" ad avviarsi, presumibilmente per una serie di 
timeout che si verificano lanciando i servizi: se faccio boot in 
modalità verbosa, ogni voce dell'avvio ci mette una vita a scorrere.
I servizi e demoni LDAP sono tutti configurati e funzionanti. Appena 
apporto questa modifica, salta tutto, devo riavviare con il CD e fare il 
recovery del file a mano.

Capisco che le informazioni che ho dato sono poche, ma posso fornire 
tutto il necessario appena si sospetta quale sia il problema.

Grazie in anticipo.
 Alessio
