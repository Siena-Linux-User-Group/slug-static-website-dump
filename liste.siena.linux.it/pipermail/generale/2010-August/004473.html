Bhe se stai paralndo di un utente loggato in un'-interfaccia php che
ti carica un file sul server, allora l-id glielo passi tu
semplicemente come parametro dell'applicazione
Esattamente come gli passi il file via post con i vari attributi size
name etc, alla form ci aggiungi un campo input, magari di tipo hidden,
dove specifichi l'id dell'utente che vuoi...

Se invece ti rifewrisci ala "nuda" funzione del php per caricare un
fiel dal tuo pc a un server, cioè move_uploaded_file che ti porta il
file dalal dir temporanea su quella che vuoi, non mi risulta che tra i
parametri dell'array $_FILES che lui usa ci sia l'utente del sistema
(apache, httpd, nobody...) che fa questa azione, ma non credo che ti
interessi sapere se il file viene caricato da quale utente del server
che stai usando. Bensi appunto è interessante sapere quale utente sta
usando l'interfaccia di upload che hai fatto in php.
Appunto come sopra, ci aggiungi un campo al post e te lo leggi al
momento di fare il move_uploaded_file...
